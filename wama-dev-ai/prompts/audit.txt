## ARCHITECTURAL RULES (from CLAUDE.md — always apply)

### HuggingFace model integration — mandatory pattern:
- Path declared in settings.MODEL_PATHS
- DIR constant in model_config.py
- HF_HUB_CACHE + HUGGINGFACE_HUB_CACHE set BEFORE any transformers/diffusers import
- cache_dir passed to from_pretrained()
- Model registered in model_registry.py _discover_*_models()
Mnemonic: "Path first, env vars second, import after"

### Common UI rules — all queue-based apps:
Every app with a processing queue MUST have:
- Per-item duplication button (using duplicate_instance() from wama/common/utils/queue_duplication.py)
- Per-item deletion button
- Per-item start button (unless auto-processed on upload)
- Global "Start all" and "Clear all" buttons
- Individual result download
- Batch import (for URL-input or text-input apps)
Missing: Composer app is currently non-compliant (no duplication button).

### Static files rule:
Modified JS/CSS files in wama/<app>/static/ MUST be copied to staticfiles/<app>/.
Python files (views.py, tasks.py, utils/) do NOT need this copy.

### Celery tasks:
- GPU-intensive tasks must use the dedicated GPU queue
- select_for_update() in transaction.atomic() to prevent race conditions on task start
- Revoke old task_id before dispatching a new one for the same item
