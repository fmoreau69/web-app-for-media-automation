## ⚠️ Discipline GIT (leçon de la session — CRITIQUE)

**4 instances tournent en parallèle et commitent seules.** Un `git add <fichier> && git commit` a
**balayé 12 fichiers stagés par une autre instance**. Règles :
- **Toujours** `git commit <chemins explicites> -m "…"` — jamais `git add .` / `git add -A` /
  `git add <un fichier> && git commit` (le commit emporte TOUT l'index partagé).
- `git status` avant de commiter ; ne commiter que ce que TU as touché.
- **Partitionner** le travail par sous-système (deux instances ne touchent jamais le même fichier).
- Solution robuste recommandée : **un `git worktree` par instance** (index + working dir isolés) →
  ni sweep d'index ni clobber de fichier. Merge maîtrisé vers dev.
- `PROJECT_STATUS.md` est édité en concurrence → éditer par petits blocs, relire avant chaque edit.
